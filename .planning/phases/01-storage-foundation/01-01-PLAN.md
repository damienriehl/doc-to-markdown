    Create `src/projectSerializer.js` with the following exact structure:

    **Exports:**
    - `export const SCHEMA_VERSION = 1;`
    - `export function serializeProject({ id, name, book, chapters, uiState = {}, createdAt })`
    - `export function deserializeProject(projectRecord, blobMap = new Map())`

    **serializeProject implementation:**
    1. Set `const now = new Date().toISOString();`
    2. Initialize `const blobs = [];`
    3. Map over `chapters` to produce `serializedChapters`:
       - For each chapter, compute `const blobId = chapter.blobId ?? crypto.randomUUID();`
       - If `chapter.file` exists (truthy), push `{ id: blobId, file: chapter.file, name: chapter.fileName }` to `blobs`
       - Return a plain object with these exact fields:
         ```
         id: chapter.id,
         blobId,
         fileName: chapter.fileName,
         fileType: chapter.fileType,
         title: chapter.title,
         slug: chapter.slug,
         chapterNum: chapter.chapterNum,
         topics: chapter.topics ?? [],
         keyTerms: chapter.keyTerms ?? [],
         markdownContent: chapter.markdownContent ?? "",
         status: chapter.status ?? "pending",
         ```
       - NOTE: `chapter.file` is deliberately omitted from the returned object (this is the whole point)
    4. Build `projectRecord`:
       ```
       {
         id,
         name,
         version: SCHEMA_VERSION,
         createdAt: createdAt ?? now,
         updatedAt: now,
         book: { title: book.title ?? "", author: book.author ?? "" },
         chapters: serializedChapters,
         uiState,
       }
       ```
    5. Return `{ projectRecord, blobs }`

    **deserializeProject implementation:**
    1. Map over `projectRecord.chapters ?? []` to reconstruct chapters:
       - Spread all chapter fields
       - Set `file: blobMap.get(chapter.blobId) ?? null`
       - Ensure `keyTerms: chapter.keyTerms ?? []` and `topics: chapter.topics ?? []`
    2. Return:
       ```
       {
         book: projectRecord.book ?? { title: "", author: "" },
         chapters,
         uiState: projectRecord.uiState ?? {},
       }
       ```

    **JSDoc:** Add `@param` and `@returns` annotations matching the existing serverApi.js pattern. Include a file-level comment explaining the serialization boundary (File objects collapse to {} under JSON.stringify).

    **Section separators:** Use `// --- Schema ---------` and `// --- Serialize ---------` and `// --- Deserialize ---------` to match existing codebase convention.
