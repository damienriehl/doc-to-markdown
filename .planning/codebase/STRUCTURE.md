## Directory Purposes

**Root (Python/Config):**
- **`convert.py`** — Main CLI entry point. Orchestrates the batch conversion pipeline:
  - Discovers source files with format deduplication
  - Routes to appropriate converter (Pandoc/Marker/PyMuPDF4LLM/browser for TXT)
  - Calls post-processor for cleanup and YAML injection
  - Triggers index generation
- **`postprocess.py`** — Shared post-processing module. Used by both `convert.py` (line 345) and `server.py` (line 91):
  - `PostProcessor.clean()` — removes conversion artifacts
  - `detect_and_promote_headings()` — promotes outline text to Markdown headings
  - `inject_yaml_header()` — adds YAML front matter
  - `extract_numbering_from_docx()` — reads DOCX numbering levels for heading inference
- **`generate_index.py`** — Generates `00-index.md` with chapter overview table and cross-reference scaffold
- **`validate.py`** — Checks converted output files for common issues (completeness, format, encoding)
- **`server.py`** — Optional FastAPI server (port 9378 default). Provides `/health` and `/convert` endpoints for web UI. Reuses `convert_buffer()` from conversion logic.
- **`chapters.yaml`** — Configuration file mapping source filenames to chapter metadata (number, slug, title, topics, key_terms)
- **`setup.sh`** — Installer for system dependencies (Pandoc, Marker) and Python packages

**src/ (Web App):**
- **`main.jsx`** — React DOM entry point. Mounts `App` component to `#root`.
- **`App.jsx`** — Main UI component (~950 lines):
  - File input: drag-drop files/folders (via `resolveDataTransferItems()`), file picker
  - Chapter inference: applies 7 strategies to extract chapter numbers from filenames (`inferChapterNum()`)
  - Batch consensus: finds dominant chapter numbering strategy across files (`applyBatchConsensus()`)
  - Conversion orchestration: routes to Mammoth (DOCX), browser converters (RTF/ODT, basic quality), or API server (PDF/RTF/ODT, full quality)
  - Download generation: creates markdown files for download
- **`inputResolver.js`** — Unified input normalization layer:
  - `isSupportedFile()` — checks extension against whitelist (DOCX/PDF/RTF/ODT/TXT)
  - `extractZip()` — unzips files with bomb protection (500MB max, 1000 entries max)
  - `resolveDataTransferItems()` — handles drag-drop with folder traversal (Chrome pagination workaround)
  - `resolveInputs()` — orchestrates ZIP extraction and file filtering
- **`convertRtf.js`** — Basic RTF converter (browser-side):
  - `stripRtf()` — regex-based RTF control sequence removal, special character handling
  - Returns basic Markdown (no formatting, no outline detection)
  - Quality indicator: `isBasicQuality: true`
- **`convertOdt.js`** — Basic ODT converter (browser-side):
  - Uses JSZip to unzip ODT (which is a ZIP with XML content)
  - Parses content.xml via DOMParser
  - Extracts text and basic structure
  - Quality indicator: `isBasicQuality: true`
- **`serverApi.js`** — Auto-detection and communication with optional local server:
  - `isServerAvailable()` — health check to `http://127.0.0.1:9378/health`, caches 30s
  - `convertViaServer()` — multipart form upload to `/convert` endpoint
  - Used by App.jsx to decide routing (server-side for high quality if available)
- **`index.css`** — Styling for the web UI

**src/__tests__/:**
- **`inputResolver.test.js`** — Tests for input normalization:
  - File extension validation
  - ZIP handling and safety
  - File filtering
- **`convertRtf.test.js`** — Tests for RTF converter:
  - Control sequence stripping
  - Special character handling
  - Paragraph preservation

**public/:**
- **`favicon.svg`** — Browser tab icon

**dist/:**
- Built web app (generated by Vite from `src/`). Currently committed for hosting (GitHub Pages or similar).

## Key File Locations

**Entry Points:**

- **CLI:** `convert.py:main()` (line 295)
  - Usage: `python convert.py --input-dir ./source --output-dir ./output [--pdf-engine marker|pymupdf] [--config chapters.yaml]`
  - Returns: Markdown files in output dir, plus index file
- **API Server:** `server.py:main()` (line 144)
  - Usage: `python server.py [--port 9378] [--host 127.0.0.1]`
  - Endpoints: `GET /health`, `POST /convert` (file upload)
- **Web App:** `src/main.jsx` → React mounts `App` component to browser DOM
  - Accessed via browser at file:// or HTTP server

**Configuration:**

- **Chapter metadata:** `chapters.yaml` (lines 1–103)
  - Format: YAML with book metadata and chapter array
  - Fields: source, chapter, slug, title, topics, key_terms
  - Used by: `convert.py:load_config()` (line 25), `convert.py:get_chapter_config()` (line 127)
- **Build config:** `vite.config.js` (5 lines, minimal)
- **Node dependencies:** `package.json` (23 lines)
- **Python dependencies:** Listed in `setup.sh` (pip/conda commands)

**Core Logic:**

- **Conversion routing:** `convert.py:main()` (lines 364–376, switch on file extension)
- **Format deduplication:** `convert.py:find_source_files()` (lines 54–65)
- **ZIP bomb protection:** `convert.py:_extract_zip()` (lines 81–87) and `src/inputResolver.js:extractZip()` (lines 61–75)
- **Heading detection:** `postprocess.py:detect_and_promote_headings()` (lines 121–209)
- **Chapter inference:** `src/App.jsx:inferChapterNum()` (lines 37–83)
- **Server detection:** `src/serverApi.js:isServerAvailable()` (lines 18–36)

**Testing:**

- **Input resolver tests:** `src/__tests__/inputResolver.test.js`
- **RTF converter tests:** `src/__tests__/convertRtf.test.js`
- **Run tests:** `npm run test` (via Vitest)
- **Python unit tests:** Not currently present (validation via `validate.py` CLI tool)

## Naming Conventions

**Files:**

- **Python CLI tools:** Lowercase, underscores, descriptive verbs
  - `convert.py`, `postprocess.py`, `generate_index.py`, `validate.py`, `server.py`
- **Web components:** PascalCase for JSX files, camelCase for utilities
  - `App.jsx`, `convertRtf.js`, `inputResolver.js`, `serverApi.js`
- **Configuration:** Lowercase, hyphens or dots
  - `chapters.yaml`, `vite.config.js`, `package.json`
- **Output files:** Zero-padded chapter number, lowercase slug, `.md` extension
  - `00-index.md`, `01-trial-preparation.md`, `02-motion-practice.md`
- **Test files:** Co-located with source, `.test.js` suffix
  - `src/__tests__/inputResolver.test.js` (mirrors `src/inputResolver.js`)

**Directories:**

- **Semantic folders:**
  - `src/` — All web app source code
  - `src/__tests__/` — Unit tests for web modules
  - `public/` — Static assets (favicon, etc.)
  - `docs/` — Planning and brainstorms
  - `dist/` — Built/compiled output (generated)
- **Avoid:** Generic names like `lib/`, `utils/` (use semantic module names instead)

## Where to Add New Code

**New Conversion Format (e.g., `.docm` or `.epub`):**

1. **CLI side:**
   - Add extension to `FORMAT_PRIORITY` dict in `convert.py` (line 35)
   - Implement `convert_docm()` function in `convert.py` with signature matching existing converters (return bool, write to output path)
   - Add case in `convert.py:main()` switch (around line 364–376)
   - Update `chapters.yaml` example comments
   - Update `setup.sh` if new system dependency is required

2. **Web side:**
   - Add extension to `SUPPORTED_EXTENSIONS` set in `src/inputResolver.js` (line 8)
   - If simple text extraction: implement `src/convertDocm.js` export `convertDocm(file)` returning `{ md, ... }`
   - If complex: route through API server only (document in code comments)
   - Update `ALLOWED_EXTENSIONS` in `server.py` (line 45)

**New Post-Processing Rule:**

1. Add regex pattern and transformation to `PostProcessor.clean()` in `postprocess.py` (lines 56–87)
2. Document pattern and why it's needed in code comment
3. Add test case to verify the rule works (in `src/__tests__/` or via manual test in `convert.py` doctest)
4. If pattern is complex, extract to separate method (following pattern of `detect_and_promote_headings()`)

**New Chapter Inference Strategy:**

1. Add strategy to `inferChapterNum()` in `src/App.jsx` (after line 43)
2. Return object: `{ num: <int>, strategy: "<name>", confidence: <0–1> }`
3. Update `applyBatchConsensus()` if strategy needs special voting logic
4. Add test case to `src/__tests__/` (new file if needed)

**New Cross-Reference in Index:**

1. Edit `CROSS_REFERENCES` array in `generate_index.py` (lines 17–54)
2. Add concept object with locations array
3. Run `python generate_index.py` to regenerate index (if not using `convert.py --skip-index`)

**New Environment Configuration:**

1. Add to `chapters.yaml` (book metadata section) or create separate config file
2. Load in `convert.py:load_config()` or `server.py` startup
3. Document in `CLAUDE.md` and `README.md`
4. Never commit secrets to git (use `.env` pattern and `.gitignore`)
